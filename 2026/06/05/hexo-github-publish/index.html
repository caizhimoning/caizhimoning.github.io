Hexo 博客发布 GitHub 与自动化

发表于2026-06-05
|浏览量:

项 要求

概念 git init / push / .gitignore 见 Git 基础：版本管理与常用操作

双仓库分工

仓库 可见性 内容 推送方式
caizhimoning.github.io Public hexo g 生成的静态页（HTML/CSS/JS） hexo d（hexo-deployer-git）
my-blog-source Private Hexo 源码：source/、_config*.yml、主题、package.json git push

hexo d 仅更新公开 Pages 仓库，不会向私有库写入内容；源码备份须单独执行 git push 或 npm run git-backup。

站点 URL 以 _config.yml 为准（当前为 root: /project/，访问路径含 /project/）。

一、GitHub：创建公开仓库

用于 GitHub Pages 展示编译后的站点。

  1. 登录 GitHub → 右上角 + → New repository。
  2. Repository name：caizhimoning.github.io（须与用户名一致、全小写）。
  3. Visibility：Public（免费账号 Pages 通常要求公开仓库）。
  4. Initialize：不勾选 README、.gitignore、license（保持空仓库，便于首次 hexo d 覆盖推送）。
  5. 点击 Create repository。
  6. 创建完成后无需在网页执行 push，后续在 WSL 中操作。

二、GitHub：添加 SSH 公钥

供 WSL 内 hexo d 与 git push 使用 SSH 认证。

  1. 头像 → Settings → SSH and GPG keys → New SSH key。
  2. Title：标识本机（如 WSL2-Blog）。
  3. Key：粘贴 WSL 中 cat ~/.ssh/id_ed25519.pub 的完整输出。
  4. Add SSH key。

公钥生成见第四节 4.1。

三、GitHub：创建私有仓库

用于备份 Markdown 与 Hexo 配置。

  1. New repository。
  2. Repository name：my-blog-source（名称可自定，须与下文 remote 一致）。
  3. Visibility：Private。
  4. 不勾选初始化选项 → Create repository。
  5. 记录 SSH 地址：git@github.com:caizhimoning/my-blog-source.git。

八、故障处理

现象 处理
部署状态异常 rm -rf .deploy_git 后重新 hexo clean && hexo g && hexo d
验证 SSH 与公开库 git ls-remote git@github.com:caizhimoning/caizhimoning.github.io.git
hexo d 成功但私有库无更新 执行 git push 或 npm run git-backup
推送要求输入密码 ssh -T git@github.com；检查 Settings 中 SSH key
staticrypt: command not found npm install -g staticrypt
missing script: publish 检查 package.json 中 scripts.publish

公开库中的 HTML 无法还原为 Markdown；源码以私有库为准。
