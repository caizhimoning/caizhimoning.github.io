Zotero 使用指南

发表于2026-06-05
|浏览量:

Zotero 是开源文献管理软件，用于收集、组织 PDF 与元数据，并在 Word、LibreOffice 或 LaTeX 中插入引用与生成参考文献表。本文覆盖桌面端与浏览器扩展的安装、常用导入方式、分类与标签、PDF 阅读标注、引用写作及同步备份；LaTeX 排版见 LaTeX 语法入门。

Zotero 是什么

能力 说明
抓取 浏览器扩展从网页、数据库一键保存条目与 PDF
管理 本地库 + 可选云端同步；集合、标签、全文搜索
阅读 内置 PDF 阅读器，高亮与批注可写入条目
引用 Word / WPS / LibreOffice 插件；导出 BibTeX 供 LaTeX

数据默认保存在本机 数据目录（含 zotero.sqlite 与 storage/）；注册 zotero.org 账号后可使用官方同步（有免费容量上限）。

安装与账号

客户端

  1. 打开 https://www.zotero.org/download/ 下载 Windows / macOS / Linux 安装包。
  2. 安装后启动 Zotero，菜单 编辑 → 首选项（macOS 为 Zotero → Settings）中可改界面语言与数据目录。
  3. Linux 亦可使用发行版包或 tarball；确保有 LibreOffice 集成时需额外安装对应组件。

浏览器扩展

扩展名 浏览器
Zotero Connector Chrome、Firefox、Edge、Safari 等

安装 Connector 后，访问期刊页、知网、arXiv、Google Scholar 等时地址栏或页面会出现 保存到 Zotero 图标；按条目类型自动识别元数据。

注册与同步

  1. zotero.org 注册账号。
  2. 客户端 编辑 → 首选项 → 同步：填写用户名与密码，勾选 同步文献库数据。
  3. 文件同步 可选：
    • Zotero 存储：简单，免费账户约 300 MB 附件空间（以官网为准）；
    • WebDAV：自建或购买 WebDAV，同步 PDF 等大文件。

本库 WebDAV 示例（中科院数据胶囊）：

项 值
门户 https://aai.cstcloud.net/（创建空间）
连接方式 WebDAV
地址 https://data.cstcloud.cn/dav
配置位置 Zotero 设置 → 同步 中填写用户名与密码

首次同步建议在网络稳定时进行；多设备使用同一账号即可合并库。

插件与扩展

客户端与 Connector 装好后，通过插件扩展中文抓取、翻译、LaTeX 导出与笔记等功能。下文为本机 已安装插件 及作用；新插件可用 Add-on Marketplace 搜索安装。

安装方式

方式 步骤
Add-on Marketplace（推荐） 先安装该插件 → 菜单 工具 → Add-on Marketplace → 搜索 → 安装 / 更新
手动 .xpi 工具 → 插件 → 齿轮 → Install Add-on From File → 选择 .xpi
官方索引 Zotero 插件文档 查找开发者发布的安装包

安装或升级后 完全退出并重启 Zotero；大版本升级前建议备份数据目录。部分插件依赖 Better BibTeX 等基础组件，需按提示一并安装。

已安装插件一览

插件 作用
Add-on Marketplace 在 Zotero 内浏览、安装、更新与查看社区插件（本库其余插件多由此安装）
Better BibTeX for Zotero 稳定 citekey、自动导出 .bib、与 LaTeX / Overleaf 协作；见 [LaTeX / Overleaf](#LaTeX / Overleaf)
Jasminum（茉莉花） 中文元数据、知网 / 万方等抓取与题名拆分；配合 Connector 使用
Translate for Zotero 翻译 PDF 选段、网页、元数据、批注等（多引擎可配置）
Zotero Pdf2zh PDF 全文翻译（生成双语对照 PDF 等，适合精读外文）
Better Notes for Zotero 笔记工作流增强：模板、同步、与条目深度绑定
Easier Citation 简化 Word / 编辑器中的引用插入流程
Ethereal Style for Zotero Ethereal 系列：界面与阅读样式美化
Ethereal Reference for Zotero Ethereal 系列：引用与参考文献显示相关增强
Awesome GPT for Zotero 调用 GPT 类 API：摘要、问答、辅助阅读（需自行配置 API Key）
Actions and Tags for Zotero 自定义右键 动作 与 标签 规则，批量整理条目
Zotero Attanger 附件（PDF 等）自动归类、移动与路径管理
Linter for Zotero 检查并格式化条目元数据（作者、刊名、DOI 等字段规范）
Green Frog 根据 DOI / 刊名等 更新期刊信息（影响因子、分区等，视数据源）
Sci-PDF 尝试从 Sci-Hub 等源 自动补全 PDF；涉及版权与机构政策，请合规使用

同步设置

如何手动迁移这些设置？
如果你需要在多台电脑上保持统一，只能手动操作：

找到数据目录：在 Zotero 中点击 编辑 → 首选项 → 高级 → 文件和文件夹，点“数据目录”下的“打开数据目录”。

复制关键文件夹/文件：

插件及配置：复制整个 plugins 文件夹。

软件核心设置：复制 prefs.js 文件（包含全部偏好设置）。

在新电脑装好 Zotero 后，先关闭软件，再将这些文件覆盖到新电脑对应的数据目录中。

注意，两台电脑的 Zotero 版本和插件最好保持一致，否则直接覆盖 prefs.js 或插件文件夹可能会导致异常。如果只是迁移插件，复制 plugins 文件夹后重启 Zotero 通常就能自动加载。

文献库结构

概念 说明
条目（Item） 一篇文献的记录：类型（期刊文章、图书、会议论文等）、标题、作者、年份、DOI 等
附件（Attachment） 挂在条目下的 PDF、快照、链接；PDF 可自动重命名
笔记（Note） 富文本笔记，可与条目关联
集合（Collection） 文件夹式分类，同一文献可属于多个集合
标签（Tag） 扁平关键词，支持彩色标签与搜索
群组库（Group Library） 多人共享；个人库与群组库分开显示

左侧为 我的文库 与群组；中间为条目列表；右侧为 信息、笔记、标签、相关 等面板。双击条目可在内置阅读器中打开 PDF。

导入文献

方式 步骤 / 备注
浏览器抓取 打开文献页 → 点击 Connector 图标 → 选择保存到哪个集合
标识符导入 工具栏 魔棒 → 输入 DOI / ISBN / PMID / arXiv ID → 自动拉取元数据
拖入 PDF 将 PDF 拖入 Zotero；可自动识别 DOI 并补全元数据（视 PDF 质量）
从文件导入 文件 → 导入 → 选择 BibTeX、RIS、EndNote XML、CSL JSON 等
剪贴板 复制 DOI 列表等，部分场景可用插件批量导入

中文数据库（知网、万方等）常需 机构权限 或 Jasminum 才能稳定抓取；无 PDF 时可先保存元数据，日后用 Sci-PDF 或手动补附件。

元数据核对

导入后检查 条目类型、作者姓/名、刊名、卷期页；错误可在右侧 信息 面板手工修改。期刊文章建议保留 DOI 或 URL 便于后续更新。

组织：分类、集合与标签

方式 用途
按项目建集合 如「学位论文」「课程 A」；子集合表示章节或主题
按文献类型 综述 / 方法 / 待读；可用顶层集合或标签区分
标签 #待读、#已引用；支持搜索框 tag:待读
彩色标星 右键条目设置颜色，快速视觉筛选
保存的搜索 高级搜索条件存为虚拟集合，动态更新

/moning
标签可以通过拖拽快速赋予，也可以赋予颜色和数字，赋予数字后，选中对应文献，输入数字即可快速赋予标签
集合可以按照读文献的方式来，边读边创建引用笔记。最后统一生成一个参考文献引用集合。

阅读与 PDF 标注

  1. 双击 PDF 在内置阅读器打开（Zotero 6+ 统一阅读器）。
  2. 工具栏：高亮、下划线、笔记、区域截图 等。
  3. 标注自动生成 子笔记，挂在对应条目下，可导出或用于回顾。
  4. 右键标注 → 创建独立笔记 可汇总摘录。
操作 说明
搜索全文 编辑 → 搜索，或库内搜索框限定当前 PDF
旋转 / 适应宽度 阅读器右下角控件
外部阅读器 右键附件 → 用系统默认程序打开（标注不同步回 Zotero）

写论文时可在 Zotero 中选中文献 → 右键 → 由所选条目创建参考文献表 预览书目格式。

引用与写作

Zotero 通过 CSL（Citation Style Language） 样式控制著录格式（国标 GB/T 7714、APA、IEEE、Nature 等）。样式可在 Zotero Style Repository 下载。

/moning
下载后需要对格式进行修改

Word / WPS

  1. 安装 Zotero 时勾选 Office 插件（或首选项中重装）。
  2. Word 出现 Zotero 选项卡：Add/Edit Citation 插入引文，Add/Edit Bibliography 插入参考文献表。
  3. 首次使用选择 Document Preferences：引用样式、语言、脚注或著者-年份。
  4. 完稿前 Refresh 更新域代码；交稿前可 Unlink Citations 转为纯文本（不可逆）。

WPS 兼容性因版本而异；异常时可尝试以管理员安装插件或改用 Word / LibreOffice。

LaTeX / Overleaf

项目 说明
导出 BibTeX 右键集合或条目 → 导出文献库 → 格式选 BibTeX；或安装 Better BibTeX 自动导出 .bib
引用键 Better BibTeX 生成稳定 citekey（如 authorTitleYear）
Overleaf 上传 .bib 或在项目中同步；正文 \cite{key}
样式 LaTeX 侧用 biblatex + biber 或传统 BibTeX + .bst；与 Zotero CSL 样式可并存，需统一一种流程

同步与备份

方式 说明
官方同步 文献元数据与笔记；附件可选 Zotero 或 WebDAV
WebDAV 坚果云、Koofr、自建 Nextcloud 等；在首选项 → 同步中填写 URL 与路径
手动备份 复制整个 数据目录；或 文件 → 导出文献库 为 Zotero RDF / JSON
版本恢复 官方同步对部分条目保留版本历史（右键条目查看）

数据目录位置：
首选项 → 高级 → 文件和文件夹 → 数据目录位置

常见问题

现象 处理
Connector 灰色 / 无法抓取 确认已装桌面 Zotero 并运行；刷新页面；换 Firefox 试
Word 无 Zotero 菜单 重装插件；关闭 Word 后从 Zotero 首选项安装；检查 32/64 位一致
中文作者显示异常 条目信息中拆分「姓 / 名」；样式选 GB/T 7714；BBT 可设拼音键
同步冲突 选择保留版本；避免多设备同时离线大改后同时上线
PDF 未下载 无权限或抓取仅元数据；手动拖入 PDF 或通过机构 VPN
BibTeX 键重复 用 Better BibTeX 固定命名规则
库损坏 自备份恢复；或 zotero.sqlite 修复工具（谨慎，先备份）
