Markdown 语法入门

发表于2026-06-06
|浏览量:

Markdown 是一种易读易写的纯文本标记语言，由 John Gruber 与 Aaron Swartz 设计。语法说明以官网为准：Markdown Syntax Documentation。

本文按官网结构介绍：块级元素、行内元素、转义与其它。Obsidian、Hexo 在此基础上常支持表格、删除线等扩展（文末简要说明）。

块级元素（Block Elements）

段落与换行（Paragraphs and Line Breaks）

段落之间用空一行分隔。单换行不会产生新段落；在行末加两个空格再回车，可得到硬换行（<br>）。

标题（Headers）

两种写法。

Setext 式（一级、二级）：下划线

Atx 式（一至六级）：行首 #，数量表示级别

Closing hashes 可选：## 标题 ##。官方建议 # 与标题文字之间留空格。

引用（Blockquotes）

行首 >，可多行、可嵌套

列表（Lists）

无序：行首 -、* 或 + 加空格（三种等价）

有序：行首 数字. 加空格；数字本身不必连续，渲染时按顺序输出

列表项内可含多段：后续段落缩进四个空格；引用块在列表项内也须缩进

代码块（Code Blocks）

缩进式：每行前加四个空格或一个 Tab（与列表嵌套规则一致）

水平线（Horizontal Rules）

单独一行，仅包含 *** 或 ***** 或 --- 或 ----------（至少三个 * 或 -，可含空格）

行内元素（Span Elements）

链接（Links）

行内式：

引用式：文中写 [标签][id]，文末定义

隐式引用：标签与 id 相同时可省略第二处 id

自动链接：尖括号包住 URL 或邮箱

强调（Emphasis）

效果 写法
斜体 *斜体* 或 _斜体_
粗体 **粗体** 或 __粗体__

星号、下划线与文字之间若被空格隔开则不解析为强调。习惯上强调内部可再嵌套，以渲染器为准。

行内代码（Code Spans）

反引号 ` 包裹

若代码本身含反引号，用多个反引号包裹：`` `foo` ``

内联 HTML

官网说明：Markdown 基于 HTML，标签块级/行内若未在语法中定义，可直接写 HTML；块级 HTML 块前后需空行，且不要用空格缩进，否则会被当成代码块。
