Obsidian 设置：字体、外观与 Properties

发表于2026-06-09
|浏览量:

界面发虚、中文变 □（tofu）、Properties 显示异常，多半是 字体与外观 未配好。WSLg 下写中文需 fcitx5；本库提供 一键启动脚本（输入法 + Obsidian）。用法与双链见 Obsidian 学习与用法手册；Win/WSL 安装见 Obsidian 在 Windows 与 WSL2 中的安装与库路径选择。

一、中文方框：安装系统字体（WSL / Linux）

Properties 里 title、subtitle 等出现方框，说明 系统缺少 CJK 字体，Obsidian 界面字体无法渲染汉字。

在 WSL Ubuntu 终端执行：

安装后必须：

  1. 完全退出 Obsidian（不是只关窗口）
  2. 重新执行 obsidian "/home/moning/obsidian_dir"
  3. 仍异常时：wsl --shutdown，再进 WSL 后重开 Obsidian

验证字体是否可用

应能看到 Noto Sans CJK SC、Noto Serif CJK SC 等。

二、标题 Emoji 变方框（如 welcome-to-my-blog）

welcome-to-my-blog 等笔记标题里的 🌟 📝 🎯 是 Emoji，不是汉字。缺 彩色 Emoji 字体 时会显示成方框（☒）。

安装 Noto Color Emoji

安装后完全退出并重启 Obsidian。验证：

本库 appearance.json 已在界面/正文/代码字体栈末尾加入 Noto Color Emoji 作为回退。

若仍不显示

  1. 设置 → 外观 → 字体：在正文字体栈中确认系统能选到 Noto Color Emoji（部分版本仅走系统 fontconfig，列表里可能不显示属正常）
  2. 或把标题改成纯文字，例如 ## 关于这个博客（去掉 Emoji），Hexo 与 Obsidian 均不依赖 Emoji 字体

三、本库 appearance.json

路径：.obsidian/appearance.json（已写入，一般无需手改）

项 当前值 作用
theme obsidian 内置浅色/深色主题名
baseFontSize 16 正文字号
interfaceFontFamily Noto Sans CJK SC, … 侧栏、菜单、Properties
textFontFamily Noto Serif CJK SC, … 编辑区正文
monospaceFontFamily JetBrains Mono, Noto Sans Mono CJK SC 代码块、行内代码

若界面仍无中文，在 Obsidian 内：设置 → 外观 → 字体，三项分别手动选 Noto Sans CJK SC（界面）、Noto Serif CJK SC（正文）、Noto Sans Mono CJK SC（代码）。

可选：安装 JetBrains Mono

未安装时代码字体列表会回退到 Noto Mono CJK，不影响中文。

四、Properties 视图 vs 源码

Obsidian 1.4+ 默认把 YAML front matter 渲染为 Properties 表单（键值列表）。

模式 设置路径 适合
表单（默认） 编辑器 → 文档中的 Properties → 显示 填 title、tags 等
源码 同上 → 源码（Source） 与 Hexo、Git diff 对齐

Hexo 发布读的是 YAML 文本，两种视图数据相同。写博客 front matter 时若需对照模板，可临时切源码。

常用 Properties 类型

  • 文本：title、subtitle
  • 列表：tags（每项一行）
  • 数字：sticky
  • 布尔：comment（勾选框）

在表单视图点属性名旁 + 可新增字段；与 [[Templates/post-knowledge]] 字段保持一致即可。

五、主题与观感

内置主题

设置 → 外观 → 主题：Obsidian（默认）、Moonstone 等，可切换浅色/深色。

社区主题

  1. 设置 → 外观 → 主题 → 管理
  2. 搜索并安装，例如：Minimal、Blue Topaz、Things
  3. 启用后可在「CSS 代码片段」里微调（进阶）

社区主题可能影响 Properties、表格样式；若与 Hexo 预览差异大，写作时以 默认主题 + 本库字体 为准。

与博客视觉的区别

博客「樱花风」来自 Hexo Butterfly，与 Obsidian 编辑器无关。Obsidian 只负责 写作体验；发布后在浏览器看最终版。

七、WSLg 中文输入法与一键启动

WSL 内 Obsidian 是 Electron/GTK 应用，不能直接用 Windows 输入法。本库用 fcitx5 + 拼音，并通过脚本在启动 Obsidian 前自动拉起输入法。

脚本一览

脚本 作用 何时运行
scripts/install-chinese-ime-wsl.sh 安装 fcitx5、拼音、GTK/Qt 前端；写入 ~/.config/fcitx5 与 ~/.bashrc 环境变量 一次性（需 sudo）
scripts/start-fcitx5-wsl.sh 启动 fcitx5（已运行则跳过） 单独调试输入法，或由一键脚本调用
scripts/obsidian-wsl.sh 一键启动：fcitx5 → Obsidian 日常

scripts/install-wsl-browser-bridge.sh Linux 点链接 → Windows Chrome（wslview） 一次性（需 sudo）

Linux 应用打开网页链接（Obsidian 点 source 等）详见 WSL 常用设置：Linux 应用打开 Windows 浏览器。

1. 安装输入法（一次性）

在 WSL 终端执行：

脚本会：

  • 安装 fcitx5、fcitx5-chinese-addons、GTK/Qt 前端等
  • 在 ~/.config/fcitx5/conf/wayland.conf 中 禁用 Wayland 模块（WSLg 下否则 fcitx5 易秒退）
  • 默认输入法组：美式键盘 + 拼音
  • 在 ~/.bashrc 写入 GTK_IM_MODULE=fcitx 等（不会每次开终端自动启动 fcitx5）

安装后按提示验证：

在 Obsidian 编辑区按 Ctrl+Space 切换中/英。

2. 日常一键启动

默认打开库 /home/moning/obsidian_dir；指定其它库：

obsidian-wsl.sh 依次执行：

  1. start-fcitx5-wsl.sh — 启动输入法（fcitx5 --disable=wayland --disable=waylandim）
  2. 若已装 wslview 则设置 BROWSER=wslview（点链接用 Windows Chrome）
  3. 设置 GDK_BACKEND=x11、ELECTRON_OZONE_PLATFORM_HINT=x11 等，再 exec obsidian

不要在已配好输入法的环境里单独执行裸 obsidian .，否则可能缺 fcitx5 或环境变量不全。

若将 Obsidian 加入 PATH（~/.local/bin/obsidian），该命令是 CLI 小工具，不能用来启动窗口；会报 The CLI is unable to find Obsidian。obsidian-wsl.sh 已改为调用 /opt/Obsidian/obsidian 主程序。

八、排错清单

现象 处理
全局中文方框 装 fonts-noto-cjk，重启 Obsidian
仅 Emoji（🌟📝）方框 装 fonts-noto-color-emoji，重启 Obsidian
仅代码块方框 装 fonts-jetbrains-mono 或改 monospace 为 Noto Sans Mono CJK SC
Properties 键名英文、值方框 同上；检查界面字体
Obsidian 里打不出中文 先 install-chinese-ime-wsl.sh，再用 obsidian-wsl.sh 启动；Ctrl+Space 切到拼音
fcitx5 启动后秒退 确认 ~/.config/fcitx5/conf/wayland.conf 中 Enabled=False；用 start-fcitx5-wsl.sh 启动
保存后脚本报错 ^M 勿用 Windows 版 Obsidian 打开 WSL 库；见 Obsidian 在 Windows 与 WSL2 中的安装与库路径选择
Ctrl+V 贴不进截图 用 Win+Shift+S 存 notes/attachments/ 后拖入；或试 install-clipboard-bridge-wsl.sh
Obsidian 点链接无反应 见 WSL 常用设置：Linux 应用打开 Windows 浏览器：install-wsl-browser-bridge.sh + obsidian-wsl.sh
主题安装失败 检查网络；国内可稍后重试或用手动 CSS 片段
