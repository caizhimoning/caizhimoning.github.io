Obsidian 学习与用法手册

一、核心概念

1.1 库（Vault）

  • 库 = 磁盘上的一个文件夹，内含全部 .md 与 .obsidian/ 配置。
  • 本库路径：/home/moning/obsidian_dir（WSL 内用 Linux 路径打开，见 Obsidian 在 Windows 与 WSL2 中的安装与库路径选择）。
  • 启动：obsidian "/home/moning/obsidian_dir"。

1.2 笔记与链接

  • 一篇笔记 = 一个 .md 文件。
  • 内部链接默认用 Wiki 语法 [[笔记名]]，保存为纯文本，不依赖数据库。
  • 双向：A 链到 B 时，B 的「反向链接」面板会看到 A。

1.3 库内链接 vs 博客链接

场景 写法 示例
Obsidian 库内 [[...]] [WSL2 安装与首次配置](/2026/06/05/wsl2-install/)
Hexo 发布后 站内绝对路径 /2026/06/03/wsl2-install/

定稿发博客前，正文中的 [[...]] 需改为 Hexo 路径（或保留仅库内用的链接）。详见 [[Templates/publish-checklist]]。

二、库与本工作流

2.1 目录（扁平 + MOC，不镜像博客）

目录 用途
00-inbox/ 随手记，可无 front matter
notes/ 全部正文，扁平存放
moc/domain/ 领域 MOC
moc/series/ 系列 MOC，用双链聚合笔记
Templates/ 发布用 YAML 模板

my_blog/source/_posts/hexo-butterfly/ 等只在博客仓库；用 领域 MOC + 系列 MOC（moc/domain/、moc/series/）组织，不建同名博客子目录。

2.2 推荐写作流程

对照表：领域总览、系列总览。

2.3 打开与切换笔记

  • Ctrl+O：快速切换（输入标题模糊匹配）。
  • 文件列表：左侧资源管理器，点击即打开。
  • Ctrl+点击 链接：新面板打开目标笔记（默认行为可在设置中改）。

三、交叉引用

3.1 链接整篇笔记

  • 输入 [[ 触发自动补全。
  • 路径含子目录：[[moc/series/WSL2]]。
  • 仅文件名：[Obsidian 学习与用法手册](/2026/06/04/obsidian-guide/) 对应 notes/obsidian-guide.md。

Ctrl / Cmd + 点击 跳转；悬停可预览（需开启「页面预览」插件）。

3.2 链接到标题

# 后为标题文字（Obsidian 会模糊匹配级别）。标题改名后，可用「更新内部链接」批量修复（设置 → 文件与链接）。

3.3 嵌入（Transclusion）

  • ![[笔记]]：嵌入整篇笔记全文。
  • ![[笔记#标题]]：嵌入该标题下内容。
  • 源笔记修改后，嵌入处自动同步。

适合：MOC 汇总待办、在总览页显示子页片段。

3.6 反向链接、出链、未链接提及

右栏 反向链接：

  • 链接：已用 [[ ]] 链入当前笔记的页面。
  • 未链接提及：正文出现当前笔记标题但未加链接，可一键转为链接。

出链：当前笔记指向哪些笔记。

四、命令面板与快捷键

Ctrl+P 打开命令面板，输入英文或中文关键词过滤。

4.1 常用命令

命令（关键词） 作用
Insert template 插入 Templates/ 下模板
Toggle edit / reading 切换编辑与阅读模式
Open graph view 打开关系图谱
Add internal link 为选中文本包上 [[ ]]
Follow link under cursor 跳转到光标下链接
Move file to another folder 移动笔记（少用，本库宜保持 notes 扁平）
Change theme 切换主题

4.2 快捷键速查（Windows / WSL 默认）

快捷键 作用
Ctrl+N 新建笔记
Ctrl+S 保存
Ctrl+O 快速切换
Ctrl+P 命令面板
Ctrl+E 编辑 ↔ 阅读
Ctrl+F 当前笔记内查找
Ctrl+Shift+F 全局搜索
Ctrl+G 当前笔记内下一个匹配
Ctrl+Shift+E 切换源码 / 实时预览（依模式）
Ctrl+, 打开设置

可在 设置 → 快捷键 中自定义。

五、模板与附件

5.1 模板插件

设置 → 核心插件 → 模板 → 文件夹 Templates/。

模板 用途
inbox-capture 收件箱结构
post-knowledge 技术博文 YAML
post-life 生活博文 YAML
publish-checklist 发布前检查

用法：命令面板 →「Insert template」→ 选文件。模板变量示例：{{title}}、{{date:YYYY-MM-DD}}（由 Templates 插件解析）。

5.2 日记（Daily notes）

已配置 .obsidian/daily-notes.json：日记创建在 00-inbox/，套用 inbox-capture。

命令面板 →「Open today’s daily note」。

5.3 附件

  • 默认目录：notes/attachments/（拖入图片、PDF 等）。
  • 粘贴截图：自动保存并在文中插入 ![[文件名.png]]。
  • 发博客：图片需复制到 my_blog/source/img/ 或文章 asset 目录，正文改用 Hexo 可访问路径。

六、搜索与元数据

6.1 全局搜索（Ctrl+Shift+F）

语法 含义
关键词 全文匹配
"精确短语" 整句匹配
path:notes 限定文件夹
file:obsidian 文件名含 obsidian
tag:#标签 标签（若使用 Obsidian 标签）
-排除词 不含该词

结果可「复制搜索结果」批量处理。

6.2 标签 vs Properties

方式 写法 本库建议
Obsidian 标签 正文 #标签 或 front matter 可选
Properties YAML tags: 列表 Hexo 以 YAML 为准
系列 series: + [[moc/...]] 推荐用 MOC 导航

写博客文章时以 Properties 为准；库内浏览用 双链 + MOC 即可。

6.3 Properties 与 Hexo

Hexo 必需字段见 [[Templates/post-knowledge]]：title、date、permalink、series、tags、abbrlink 等。

规则：文件名 = permalink 末段 = abbrlink（与 my_blog/scripts/fix-post-slug.js 一致）。

本库已开启（见 .obsidian/core-plugins.json）：

插件 查阅用途

出链 当前页引用了谁

社区插件（Web Clipper、Omnisearch、PDF++ 等）见 Obsidian 社区插件与本库选用。

7.1 画布（Canvas）

命令面板 → Create new canvas：节点可拖放，节点可 链接到现有 .md。

用途：规划系列结构、文章大纲；一般不发布到 Hexo。

7.2 关系图谱

命令面板 → Open graph view：

  • 拖动节点、缩放。
  • 过滤器：按路径 moc/、notes/ 筛选。
  • 以当前笔记为中心：命令「Open local graph」。

八、编辑与阅读

8.1 三种模式

模式 切换 说明
实时预览 默认 边写边渲染 Markdown
源码模式 命令 / 工具栏 纯文本编辑
阅读模式 Ctrl+E 只读，链接可点

8.2 预览与 Hexo 差异

Obsidian 支持 [[Wiki]]、嵌入等；Hexo 不支持。预览仅作写作参考，以 hexo g 生成结果为准。

8.3 大纲与折叠

  • 右栏 大纲 跳转标题。
  • 标题下 - 列表、## 等可折叠（光标在行首时点击折叠三角）。

九、MOC 与图谱

9.1 什么是 MOC

MOC（Map of Content） = 索引页，只含链接与简短说明，不替代正文。

本库示例：

  • 总表：领域总览、系列总览 → 领域 → 系列
  • 示例系列：Obsidian、Cursor

9.2 维护习惯

  1. 新笔记写完，在对应 MOC 的「库内笔记」加一行 [[文件名]]。
  2. 待写清单用 - [ ]，完成打勾或删除。
  3. 系列总览可用 ![WSL2#待写](/series/wsl2#待写/) 嵌入到周报笔记。

9.3 图谱与 MOC 分工

工具 适合
MOC 人工策展、固定阅读顺序
图谱 发现意外连接、库变大后鸟瞰

十一、常见问题

Q：Wiki 链接发到博客会变成死链吗？
A：会。发布正文请用 /series/.../、/YYYY/MM/DD/slug/。

Q：能否用 Windows 版 Obsidian 打开 WSL 库（L:）？
A：不要。见 Obsidian 在 Windows 与 WSL2 中的安装与库路径选择。

Q：中文方框？
A：见 Obsidian 设置：字体、外观与 Properties。

Q：能否把 my_blog 当库？
A：不推荐；node_modules、Hexo 配置与 Obsidian 配置会混在一起。

Q：如何用 Git？
A：obsidian_dir 与 my_blog 分开仓库/分开提交即可。
