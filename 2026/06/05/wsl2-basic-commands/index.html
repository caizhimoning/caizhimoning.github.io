WSL2 基本命令

|浏览量:

关闭、终止与注销

命令 说明
wsl --shutdown 终止全部发行版，释放内存；不删数据
wsl --terminate <名> 只停一个；不删数据
wsl --unregister <名> 永久删除该发行版及其虚拟磁盘数据，不可恢复

注销前务必先 wsl --export 备份（见下）。迁盘完整流程见 [[wsl2-install#迁移到其他磁盘]]。

导出与导入

参数 含义
export 第 1 参数 发行版名称（wsl -l -v）
export 第 2 参数 输出 .tar 路径
import 第 1 参数 导入后的实例名（可与原名相同）
import 第 2 参数 存放虚拟磁盘的目录
import 第 3 参数 .tar 文件路径
--version 2 以 WSL2 导入（建议显式指定）

--import 后默认用户常为 root，需自行配置默认登录用户，见 Microsoft 导入说明。不要在未导出备份的情况下 --unregister。

Linux → 读 Windows 盘

/mnt/c/ 对应 C 盘。其他盘符同理。

性能与权限

  • 项目与代码放在 Linux 家目录（如 ~/projects），经 /mnt/c/ 大量读写明显更慢。
  • 避免在 Windows 资源管理器里对 \\wsl.localhost\... 做大规模搜索、索引。
  • 在 Windows 侧创建的文件，Linux 里可能出现权限 / 属主不符合预期；脚本与配置尽量在 Linux 内创建。

WSL2 通过 P9 协议 把 Linux 文件暴露给 Windows，本质是网络共享式访问，不是本地 ext4 直读。

找不到 Linux 文件夹？

侧边栏没有 Linux、或 \\wsl.localhost\... 报错时，多半是因为 WSL 未运行。先打开任意发行版终端（或 wsl -d Ubuntu），再试资源管理器。

开发时的性能建议

  • 写代码优先用 VS Code + WSL 扩展（code .），走专用通道，比资源管理器拖拽更快、权限更可控。
  • 项目放在 Linux 目录（如 ~/projects），不要长期在 /mnt/c/ 上编译。

映射网络驱动器（可选）

UNC 路径 \\wsl.localhost\Ubuntu\... 较长；可映射为盘符（如 L:），方便在「另存为」对话框里选盘。

  1. 资源管理器地址栏输入 \\wsl.localhost\ 回车。
  2. 右键发行版文件夹（如 Ubuntu）→ 映射网络驱动器…。
  3. 选盘符，可勾选 登录时重新连接 → 完成。

注意：执行 wsl --shutdown 后，映射盘可能显示断开（红叉），重新启动发行版后恢复。大量小文件复制仍比本地磁盘慢。

网络相关命令

默认 NAT 模式下，WSL2 虚拟 IP 可能在重启后变化；Windows 访问 Linux 服务时常需查当前 IP 或使用 localhost 转发（Win11 / 较新版本对 localhost 有改进）。若已配置 镜像网络，IP 应与 Windows 一致，配置见 [[wsl2-install#镜像网络（可选）]]。

在 PowerShell 查看 Windows IP：ipconfig；可与 Linux 侧对照。

命令速查表

PowerShell / CMD

命令 用途
wsl -l -v 发行版列表、状态、WSL 版本
wsl -d <名> 进入指定发行版
wsl --set-default <名> 设默认发行版
wsl --shutdown 关闭全部实例
wsl --terminate <名> / wsl -t <名> 关闭指定实例
wsl --unregister <名> 删除发行版及数据
wsl --export <名> <路径.tar> 导出备份
wsl --import <名> <目录> <路径.tar> --version 2 从 tar 导入
wsl --update 更新 WSL
wsl --version 查看 WSL 版本

Linux 终端

explorer.exe . 在资源管理器中打开当前目录
code . VS Code 打开当前目录（需 WSL 扩展）
cd /mnt/c/... 进入 Windows 目录
ip addr 查看 IP（NAT 下可能变）
