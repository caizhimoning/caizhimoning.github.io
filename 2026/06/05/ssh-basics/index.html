SSH 是什么

SSH 是应用层协议，默认端口 22。客户端（如 WSL 内的 ssh）与服务器建立加密通道后，可：

  • 在远程 shell 中执行命令；
  • 传输文件（scp、sftp）；
  • 作为 Git 等工具的底层传输（如 git@github.com:...）。

与 Telnet 等明文协议相比，SSH 对传输内容加密，可抵御窃听与中间人攻击（在正确校验主机密钥的前提下）。

认证方式

方式 说明 典型场景
密码 每次连接输入远程用户密码 临时登录 VPS
公钥 本地私钥 + 服务器/服务上登记的公钥 GitHub、git push、免密登录服务器

本博客工作流采用 公钥认证：私钥保留在 WSL 的 ~/.ssh/，公钥登记在 GitHub Settings → SSH and GPG keys。

密钥对与 ~/.ssh 目录

文件 性质 是否可外传
id_ed25519（或 id_rsa） 私钥 禁止；权限应为 600
id_ed25519.pub 公钥 可粘贴到 GitHub、服务器 authorized_keys
known_hosts 已信任过的主机指纹 本地自动生成
config 可选；主机别名、指定密钥 本地配置

Ed25519 为当前推荐的密钥算法；旧系统可选用 RSA（4096 位）。

与 Git 远程地址

Git 远程 URL 有两种形式：

安全与权限

  • 私钥不得提交到 Git、不得发送他人；泄露后须在 GitHub 删除对应公钥并重新生成密钥对。
  • 私钥文件权限：
  • passphrase 可在私钥丢失时增加一层保护；自动化脚本无交互时需使用未设 passphrase 的密钥或 ssh-agent（进阶）。
  • 公钥可登记在多台服务；同一私钥对应同一公钥。

故障处理

现象 可能原因 处理
Permission denied (publickey) 未登记公钥、用错密钥、私钥权限过宽 核对 GitHub SSH keys；chmod 600 私钥；ssh -T git@github.com
仍提示 HTTPS 密码 remote 为 https:// 地址 git remote -v 改为 git@github.com:...
Host key verification failed 服务器指纹变更 检查是否连错主机；必要时编辑 known_hosts 删除旧条目
Could not resolve hostname 网络或 DNS 检查网络；代理/VPN 见 [[wsl2-install#镜像网络（可选）]]

命令速查

命令 作用
ssh-keygen -t ed25519 -C "email" 生成 Ed25519 密钥对
cat ~/.ssh/id_ed25519.pub 显示公钥
ssh -T git@github.com 测试 GitHub SSH 认证
ssh user@host 登录远程主机
scp file user@host:path 复制文件到远程
git ls-remote git@github.com:user/repo.git 测试 Git 仓库 SSH 连通
