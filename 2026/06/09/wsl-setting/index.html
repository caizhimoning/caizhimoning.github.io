WSL 常用设置：Linux 应用打开 Windows 浏览器

发表于2026-06-09
|浏览量:

WSL 里通常不装图形浏览器；Linux 应用（如 Obsidian）点 http(s):// 链接时，需把请求转给 Windows 上的 Chrome。本库用 wslview（wslu 包）做桥接。输入法与 Obsidian 一键启动见 [[obsidian-setting#七、WSLg 中文输入法与一键启动]]。

现象与原因

现象 原因
Obsidian 里点 source、YouTube 链接无反应 Linux 无默认浏览器，xdg-open 无处委托
obsidian 已加入 PATH 仍打不开网页 PATH 只解决「启动 Obsidian」，不解决「打开外链」

Chrome 实际在 Windows：/mnt/c/Program Files/Google/Chrome/Application/chrome.exe。

脚本

脚本 作用
scripts/install-wsl-browser-bridge.sh 安装 wslu、注册 Chrome、写入 ~/.bashrc 的 BROWSER=wslview
scripts/obsidian-wsl.sh 启动 Obsidian 时自动 export BROWSER=/usr/bin/wslview（若已装 wslview）

安装前验证（推荐）

按顺序在 WSL 终端试；三步都成功再跑安装脚本。

1. 直接从 WSL 唤起 Windows Chrome

Windows 应弹出 Chrome。失败则检查 Chrome 安装路径。

2. 验证 wslview（需先装 wslu）

YouTube 应在 Windows Chrome 打开。

3. 验证 xdg-open（Obsidian 同类机制）

成功说明 Obsidian 点链接也可行。

持久化安装（一次性）

验证通过后执行：

脚本会：

  1. 安装 wslu、xdg-utils
  2. 将 Windows Chrome 注册为 wslview 目标
  3. 在 ~/.bashrc 写入 export BROWSER=/usr/bin/wslview
  4. 可选：配置 wslview.desktop 为 xdg 默认浏览器
  5. 测试打开 https://www.google.com

排错

现象 处理
wslview: command not found sudo apt-get install -y wslu 或重跑 install-wsl-browser-bridge.sh
Chrome 路径不存在 改脚本中的路径，或改用 Edge：wslview -r "$(wslpath -au 'C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe')"
终端里 xdg-open 可行、Obsidian 不行 必须用 obsidian-wsl.sh 启动，或确认从已 source ~/.bashrc 的终端启动
从 Windows 快捷方式开 Obsidian 快捷方式应调用 obsidian-wsl.sh，见 [[obsidian-setting#3. Windows 桌面快捷方式（可选）]]
