Cursor 使用入门：AGENTS.md、Rules 与 WSL 全局约定

发表于2026-06-07
|浏览量:

在 Cursor 里用 AI 写笔记、生成原子知识时，需要让模型知道：本库怎么组织、博客怎么发布、以及 只能在 WSL Linux 文件系统里改文件。本文说明 AGENTS.md、项目 Rules（.mdc）、User Rules（全局） 如何配合，以及 Settings 里 Always Apply 与 globs 那一行 分别该怎么填。

一、Cursor 在本工作流中的角色

工具 作用
Obsidian 人读人写：双链、MOC、查阅
Cursor AI 协作：新建/改 notes/*.md、按模板写 front matter、更新 moc/
my_blog Hexo 构建发布（AI 仅在明确要求时修改）

典型指令（在已正确配置 Rules 后，可省略「先读 AGENTS」）：

  • 「在 notes/ 写一篇关于 xxx 的原子知识，series 为 WSL2，并更新 moc」
  • 「按 Templates/post-knowledge 新建，完成后给我 publish 命令」

二、AGENTS.md：本库的 AI 总说明

路径：/home/moning/obsidian_dir/AGENTS.md（与 notes/ 同级）。

作用

  • 集中写明：目录结构、原子知识规范、front matter、发布命令、禁止事项。
  • 由项目规则 obsidian-vault.mdc 通过 @AGENTS.md 注入每个新对话（见第三节）。
  • Cursor 也会识别根目录的 AGENTS.md；不要再整篇复制到 Settings 里当另一条重复规则。

主要内容摘要

  1. obsidian_dir 与 ../my_blog 的分工
  2. notes/ 扁平 + moc/ 双链，禁止复制 source/_posts 树
  3. series → 发布文件夹（hexo-butterfly / wsl2 / linux-basics / life）
  4. 创建原子知识的检查清单
  5. 参考笔记：obsidian-guide、obsidian-setting 等

库内 Obsidian 链接：AGENTS。

何时更新

工作流变更时 先改 AGENTS.md。obsidian-vault.mdc 只有一行 @AGENTS.md，一般无需同步改 .mdc。

三、让 obsidian_dir 每个新对话读 AGENTS

3.1 前提：工作区根目录

File → Open Folder 必须打开：

若只打开 notes/ 子目录，Project Rules 与 AGENTS.md 可能不在工作区根，新对话读不到完整约定。

3.2 核心：obsidian-vault.mdc（Always Apply）

文件：.cursor/rules/obsidian-vault.mdc

要点：

项 值
规则类型 Always Apply
globs 那一行 留空 / 不填
@AGENTS.md 把 AGENTS 全文纳入上下文，避免在 .mdc 里重复粘贴

3.3 Settings 里怎么确认

Cursor Settings → Rules, Commands → Project Rules：

规则名 类型 globs
obsidian-vault Always Apply 无
wsl-linux-filesystem Always Apply 无
hexo-blog-pipeline Apply to Specific Files 见第四节

下拉不要选成 Apply to Specific Files 来承载 AGENTS——那是常见误配来源。

3.4 验证

新开 Agent 对话（不必打开任何文件），问：

本库笔记放哪？怎么发布到博客？

期望回答包含：notes/ 扁平、不镜像 my_blog 目录树、publish-to-blog.sh、不擅自改 my_blog。

四、globs 怎么填（Apply to Specific Files）

Settings 里规则上方有一行路径模式（逗号分隔），仅当规则类型为 Apply to Specific Files 时需要填写。Always Apply 的规则不要填这一行。

本库 hexo-blog-pipeline 推荐值

模式 匹配
notes/**/*.md 原子知识正文
moc/**/*.md 系列 MOC
Templates/**/*.md front matter 模板
00-inbox/**/*.md 收件箱

编辑上述文件时，额外注入 Hexo 字段与发布提醒；新对话若只聊不写 md，仅靠 obsidian-vault + AGENTS 即可。

不要照抄的示例

界面里可能出现的 **/*.ts, src/**/*.tsx 是 TypeScript 项目示例，与 obsidian_dir 无关。

可选写法

需求 globs
仅改笔记正文 notes/**/*.md
整个库所有 md **/*.md（会连 .cursor 等也匹配，一般不推荐）
含 AGENTS 根文件 单独一条规则无法混合 Always；AGENTS 用 obsidian-vault 即可

五、规则类型对照

Cursor 每条 Project Rule 在 Settings 里可选类型（与 front matter 对应）：

UI 类型 alwaysApply globs 本库用途
Always Apply true 忽略 obsidian-vault、wsl-linux-filesystem
Apply to Specific Files false 必填 hexo-blog-pipeline
Apply Intelligently false — 本库未用（靠 description 让 AI 自选）
Apply Manually — — 聊天里 @规则名 才带入

想让新对话必读 AGENTS → 必须用 Always Apply + @AGENTS.md，不能用 Specific Files + globs。

六、常见误配与排查

现象 原因 处理
新对话不知道 notes 放哪 AGENTS 被设成 Apply to Specific Files 删重复规则；保留 obsidian-vault 为 Always Apply
globs 填了 **/*.ts 误用默认示例 改为第四节 obsidian 专用 globs，或清空（Always 规则）
打开了 notes/ 子文件夹当工作区 根目录没有 AGENTS / .cursor 重新 Open Folder → obsidian_dir
Settings 里多条 AGENTS 全文重复 手动复制 AGENTS 又建了规则 只留 obsidian-vault.mdc 引用 @AGENTS.md
AI 改 /mnt/c/... 未配 User Rules 见第七节

七、User Rules 与 wsl-linux-filesystem

路径安全需对 任意工作区 生效，用两层配置：

7.1 Project Rule（本仓库内）

.cursor/rules/wsl-linux-filesystem.mdc → Always Apply，globs 留空。

7.2 User Rules（全机所有项目）

  1. Settings → Rules → User Rules
  2. 粘贴 ~/.cursor/USER-RULES-paste-into-settings.txt 全文（纯文本，无 .mdc）
  3. 保存

规则正文（英文）禁止读写 /mnt/c/、/mnt/d/、\\wsl$ 等；只允许 /home/ 等 Linux 路径。与 Obsidian 在 Windows 与 WSL2 中的安装与库路径选择 一致。

优先级（Cursor 官方）：Team Rules → Project Rules → User Rules；路径冲突以更严格为准。

八、三层配置如何协作

层级 文件 何时生效
全局 User Rules + ~/.cursor/rules/wsl-linux-filesystem.mdc 所有项目、所有对话
项目 Always obsidian-vault.mdc obsidian_dir 每个新对话
项目 按文件 hexo-blog-pipeline.mdc 编辑 notes/moc/Templates/inbox 的 md 时

十、常见问题

Q：AGENTS.md 和 obsidian-vault.mdc 重复吗？
A：不重复。.mdc 只有「去读 @AGENTS.md」；正文只在 AGENTS 维护一处。

Q：还要在 User Rules 里再贴一遍 AGENTS 吗？
A：不需要。AGENTS 是 项目级；User Rules 只放 WSL 路径 等全机约定。

Q：hexo-blog-pipeline 和 AGENTS 冲突怎么办？
A：hexo 规则是 AGENTS 子集，编辑 md 时叠加提醒；以 AGENTS 为准。

Q：Obsidian 能打开 .mdc 吗？
A：可以当文本看；日常写作见 Obsidian 学习与用法手册。

Q：my_blog 仓库要不要同样配置？
A：my_blog 可只放 wsl-linux-filesystem.mdc（Always Apply）；AGENTS / obsidian-vault 仅 obsidian_dir 需要。
