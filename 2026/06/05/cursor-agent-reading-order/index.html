|浏览量:

一、设计目标

目标 说明
AI 为主 安装、排错、概念以模型 + 官方文档为主路径
库为参考 笔记提供边界、作者验证过的差异、可信度与跳转
正文给人 教程写完整；Agent 默认不通读长文
可控成本 不扫 wsl_base 等未索引草稿；不每任务读全库

二、Agent 读取顺序

适用于 Cursor 等 Agent 在 obsidian_dir 内代用户完成安装、配置、写作等任务。

0. 任务与边界

  • 弄清用户目标、环境（OS、是否在 WSL 内等）。
  • 遵守 AGENTS.md、WSL 路径规则；不擅自改 my_blog / 不擅自 publish。

1. 路由：系列 MOC

  • 打开相关 moc/series/*.md（及必要时 moc/domain/*.md）。
  • 读 「Agent 须知」（若已写）：该看哪篇、不该看哪篇。
  • 从「库内笔记」锁定 1～3 篇；不遍历全部笔记。

不读：00-inbox/、未进系列 MOC 的大草稿（如 wsl_base.md）、Maintenance 为 draft 的篇目（除非用户点名）。

2. 主路径：通用知识 + 联网

  • 用模型知识完成 主体方案。
  • 版本、新特性、错误码 → 联网查官方文档。
  • 库不追求覆盖全部细节；缺口在此步补齐。
