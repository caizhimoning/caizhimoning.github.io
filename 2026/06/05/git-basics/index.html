Git 是什么

Git 在本地维护一个仓库（repository），保存项目的完整变更历史。每次保存称为一次提交（commit）；可关联远程仓库（remote），将提交**推送（push）或拉取（pull）**到 GitHub 等服务器。

与「复制文件夹备份」的区别：Git 记录谁、何时、改了什么，可回退到历史版本，并支持多机协作。

gitignore

.gitignore 是仓库根目录下的文本文件，列出不应被 Git 跟踪的路径模式。

作用：执行 git add . 时，匹配项被忽略，不会进入提交，也不会被 push 到 GitHub。

本博客源码库常见条目：

原则：

  • 可再生成的大体积目录（node_modules/、public/）通常忽略。
  • 敏感信息（密码、密钥）须忽略或仅放入 Private 仓库并审慎评估风险。
  • 被忽略的文件不会出现在 git status 的待提交列表中。
