latex2word：武汉理工大学本科毕业论文 LaTeX 转 Word 工具链

latex2word

面向武汉理工大学本科毕业论文的 LaTeX → Word 转换工具链。在 WHUT-Bachelor 模板之外，用 Pandoc 将 main_word.tex 导出为 .docx，尽量保留公式、图表编号、交叉引用与 GB/T 7714 参考文献格式。

GitHub 仓库： caizhimoning/LaTeX2Word_SpecForWHUTBachelorTheses

本项目为个人开源工具，非武汉理工大学官方发布。PDF 排版仍请以学校规范与官方模板为准。

项目背景

理工类本科毕设普遍用 LaTeX 写公式和正文，但查重、协作或个别环节仍可能要求 Word。手动从 PDF 复制、或用 Word 自带公式编辑器重录，容易丢格式、丢编号、丢引用。

latex2word 把「LaTeX → Word」固化为一条可重复流水线：同一套 body/ 正文，一条线编译 PDF（main.tex + whut-bachelor.cls），另一条线导出 Word（main_word.tex + Pandoc），避免维护两份内容。

解决什么问题

痛点 本项目的做法
公式在 Word 里乱掉 Pandoc 转 OMML；后处理统一公式段落样式
图/表/式编号不一致 pandoc-crossref + 自建 Lua 过滤器，按「章.序号」重写
参考文献格式不对 citeproc + GB/T 7714 CSL；英文条目 locale 修正
版式与学校 Word 习惯差太远 reference.docx 样式模板约束标题、摘要、公式等
每次转换命令冗长 convert.sh / convert.ps1 一键入口 + YAML 配置
WSL 与 Windows 路径不一致 全程相对路径，不写盘符

适用对象与定位

适合：

  • 使用 WHUT 本科 LaTeX 模板写毕设，偶尔需要 Word 稿的同学
  • 已在 LaTeX 中写好正文，希望批量、可复现地导出 .docx 的用户

注意：

  • 转换后图片排版不会与latex完全一致
  • 封面页，声明页，目录页需在word中添加

定位： 第三方工具链，与 WHUT 官方无隶属关系；配合 WHUT-Bachelor 使用。

技术架构与流水线

组件 作用
Pandoc LaTeX 结构 → docx，数学公式 → OMML
pandoc-crossref 自动为 figure / table / equation 编号
Lua 过滤器 扫描源 .tex 建立 \label{fig:} / \label{tab:} / \label{eq:} 映射
CSL + citeproc 从 ref.bib 生成参考文献表
reference.docx 控制 Word 标题、摘要、公式段落等样式
后处理 公式 Tab 分隔编号、参考文献位置、摘要分节等

核心能力说明

交叉引用与编号

  • 按 config/default.yaml 中 chapter_files 列表顺序确定章号
  • 章内顺序计数 figure / table / equation
  • 输出格式：1.2、2.1；公式引用为 (2.1)
  • 标签约定：fig:、tab:、eq: 前缀

公式

  • 行间公式应用 Word 自定义样式 equation
  • 后处理将式号从 OMML 内提取到行尾，Tab 实现「居中公式 + 右对齐编号」

参考文献

  • 默认 CSL：font/GB-T-7714—2025.csl
  • 英文文献（langid = {english}）将「，等」改为「，et al.」
  • 参考文献块插入 \section*{References} 标题之后

仓库目录结构

路径约定

两层相对路径，禁止在 YAML 里写绝对路径：

层级 相对基准 示例
仓库级 仓库根 latex_en、config/default.yaml
项目级 项目目录（如 latex_en/） main_word.tex、body/chapter1.tex

命令行 project 与 --config 属仓库级；YAML 内 main_tex、bib、chapter_files 等属项目级。WSL 与 Windows 共用同一套相对路径。

双轨 LaTeX 源

轨道 入口 用途
PDF main.tex + whut-bachelor.cls 正式排版、封面、目录、XeLaTeX + biber
Word main_word.tex Pandoc 专用，标准 report 宏包，无模板专有命令

两条线共用 body/*.tex、ref.bib、figure/。Word 线不在 tex 内 \printbibliography，参考文献由 Pandoc --citeproc 生成。

Demo 样例项目

仓库内 latex_en/ 为演示文档（2 章 + 3 张示例图），用于验证流水线，不含真实论文内容：

章节 演示内容
Chapter 1 引用、单图、交叉引用
Chapter 2 表格、公式、子图、\eqref
Appendix 符号表
References 中英文文献各若干条

替换为自己的论文时：改 body/、figure/、ref.bib，并同步 config/default.yaml 中的 chapter_files。

已知限制

  • Word 入口须为 Pandoc 可解析的 LaTeX 子集（避免 WHUT 模板专有命令）
  • 公式 Tab 编号依赖 Word OMML 结构；复杂 align 环境建议人工抽检
  • pandoc-crossref 与 Pandoc 版本不完全一致时可能有警告，一般不影响 demo 使用
  • 中文正文专用模板（latex_zh/）尚未单独提供，可按 demo 自行扩展
